操作系统作用

  1. 资源管理

    1. 记录资源使用情况

    2. 资源分配

      1. 静态分配策略

        • 优点: 一次分配
        • 缺点: 浪费资源

      2. 动态分配策略

        举例: 页式管理系统, 需要有一个管理的数据结构(0/1表示分配与否)

    3. 资源回收策略

    4. 提高资源利用率

    5. 保护资源的使用

    6. 协调多进程的资源冲突

    7. 五大基本功能

      • 进程/线程管理
      • 存储管理
      • 文件管理
      • 设备管理
      • 用户接口

  2. 提供系统服务: 命令接口和编程接口

  3. 对硬件机器的扩展

    • 硬件系统上的第一层软件
    • 操作系统—虚拟机器

操作系统特征

  1. 并发: 处理多个同时性的活动
    • 宏观: 程序同时执行
    • 微观: CPU轮流执行
  2. 共享
    • 资源共享方式分类
      • 互斥共享
      • 同时共享
  3. 虚拟: 一个物理实体映射为若干个逻辑实体
  4. 随机: 在操作系统运行过程中会有不可预测的事件

操作系统分类

传统

  1. 批处理操作系统 Batch OS

    • 目的: 提高资源利用率, 增加作业处理吞吐量

    • 作业成批处理, 用户无法干预作业的处理过程, 程序代码以卡片的形式输入(卡片机)

    • 功能简单的计算机用作输入输出, 功能强大计算机处理作业

      • 单道批处理
      • 多道批处理

    • 实现

      SPOOLing system 在一台计算机上实现输入输出和执行, disk上面有输入井和输出井, 可以理解为disk上的两个buffer, 用于存储输入数据和输出数据.

      使用场景: 外设使用, 比如打印机

      ***SPOOLing系统的处理把独占式改为共享式

  2. 分时操作系统: 对CPU进行时间分片

    • 目标: 及时响应

  3. 实时操作系统

    • 目标: 及时响应请求, 完成处理

  4. 个人计算机操作系统

    • 目标: 界面友好、功能完善

  5. 网络操作系统

  6. 分布式操作系统: 多CPU或者多计算机联网

    • 统一操作系统, 处理能力强, 可靠性强, 具有透明性(用户不值得自己任务在哪个计算机执行)

  7. 嵌入式操作系统
